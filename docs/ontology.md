
# Ontology (`oskg`)

The Knowledge Graph uses a small local ontology called **`oskg`**, defined in `ontology/ontology.ttl`. It is combined with standard vocabularies such as FOAF, DCTerms, SKOS, PROV and OWL to make the graph more FAIR and interoperable.

* **Ontology namespace:** `http://example.org/oskg/ontology#`
* **Resource namespace:** `http://example.org/oskg/resource/`

`example.org` is used for documentation. For a final published version, it should be replaced by a persistent IRI such as `w3id.org` or `purl.org`.

## Classes

| Class                     | Meaning                                            | Reuses                    |
| ------------------------- | -------------------------------------------------- | ------------------------- |
| `oskg:Paper`              | A paper in the corpus                              | `schema:ScholarlyArticle` |
| `oskg:Person`             | An author or acknowledged person                   | `foaf:Person`             |
| `oskg:Organization`       | An acknowledged organization or funder             | `foaf:Organization`       |
| `oskg:Project`            | A grant or funded project                          | —                         |
| `oskg:Topic`              | A topic found by topic modeling                    | —                         |
| `oskg:SimilarityRelation` | A similarity link between two papers, with a score | —                         |

## Object properties

These properties connect one resource to another.

| Property                                | Connects                     | Meaning                                               |
| --------------------------------------- | ---------------------------- | ----------------------------------------------------- |
| `oskg:belongs_to_topic`                 | Paper → Topic                | Links a paper to its topic                            |
| `oskg:similar_to`                       | Paper → Paper                | Links similar papers                                  |
| `oskg:hasAuthor`                        | Paper → Person               | Links a paper to its authors                          |
| `oskg:acknowledges`                     | Paper → Person/Organization  | Links a paper to acknowledged people or organizations |
| `oskg:mentionsGrant`                    | Paper → Project              | Links a paper to a grant/project ID                   |
| `oskg:fundedBy`                         | Paper/Project → Organization | Links a paper or project to a funder                  |
| `oskg:sourcePaper` / `oskg:targetPaper` | SimilarityRelation → Paper   | Defines the two papers in a similarity relation       |

## Datatype properties

These properties store simple values such as strings, numbers or URLs.

| Property               | Range         |
| ---------------------- | ------------- |
| `oskg:hasDOI`          | `xsd:string`  |
| `oskg:hasURL`          | `xsd:anyURI`  |
| `oskg:publicationYear` | `xsd:gYear`   |
| `oskg:similarityScore` | `xsd:decimal` |
| `oskg:usedModel`       | `xsd:string`  |

## External enrichment properties

These properties are added from OpenAlex and Wikidata/SemOpenAlex.

| Property                | Used for     | Range         | Source         |
| ----------------------- | ------------ | ------------- | -------------- |
| `oskg:openAlexId`       | Paper        | `xsd:string`  | OpenAlex       |
| `oskg:venue`            | Paper        | `xsd:string`  | OpenAlex       |
| `oskg:citedByCount`     | Paper        | `xsd:integer` | OpenAlex       |
| `oskg:openAccessStatus` | Paper        | `xsd:string`  | OpenAlex       |
| `oskg:openAlexConcept`  | Paper        | `xsd:string`  | OpenAlex       |
| `oskg:wikidataId`       | Organization | `xsd:string`  | Wikidata       |
| `oskg:rorId`            | Organization | `xsd:string`  | Wikidata / ROR |
| `oskg:country`          | Organization | `xsd:string`  | Wikidata       |
| `oskg:officialWebsite`  | Organization | `xsd:anyURI`  | Wikidata       |

Enriched papers and organizations also get `owl:sameAs` links to OpenAlex, Wikidata or ROR. This connects the local Knowledge Graph to external Linked Open Data sources.

## Reused standard properties

Some general properties are represented with standard vocabularies instead of custom `oskg` terms.

| Project concept           | Represented as                          |
| ------------------------- | --------------------------------------- |
| Paper title               | `dcterms:title`                         |
| Person/organization name  | `foaf:name`                             |
| Abstract text             | `dcterms:abstract`                      |
| Topic label               | `rdfs:label` + `skos:prefLabel`         |
| Generated by / provenance | `prov:wasGeneratedBy`, `oskg:usedModel` |

## Similarity modeling

Paper similarity is represented in two ways:

| Representation                 | Purpose                                                    |
| ------------------------------ | ---------------------------------------------------------- |
| `Paper oskg:similar_to Paper`  | Simple link for easy graph navigation                      |
| `oskg:SimilarityRelation` node | Stores the source paper, target paper and similarity score |

This is needed because a normal RDF triple cannot directly store a score. The extra `SimilarityRelation` node allows the graph to keep the exact similarity value.
